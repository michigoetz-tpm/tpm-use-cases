# Contributing to TPM AI Playbook

Thanks for your interest in contributing. This repo helps Technical Program Managers work better with AI by providing execution-focused skills in an agent-ready format.

---

## Design Philosophy

Every skill is built around one principle: **the TPM stays in the loop at the judgment step**.

AI handles the overhead. The TPM owns the read on the room, the pattern behind the pattern, and the decision that moves things.

Skills should:
- Focus on execution outcomes, not documentation for its own sake
- Surface risks early and visibly
- Carry context forward across a workflow chain
- Work with whatever data the TPM has

---

## Who Can Contribute?

Anyone. If you have a TPM framework you use repeatedly, spotted a gap in an existing skill, or have feedback on what is missing — you can contribute.

---

## What Can You Contribute?

### New Skills
Good candidates: processes you run consistently, decision frameworks, communication templates that work across program types.

Not a good fit: one-off tips (use prompts instead), company-specific processes, frameworks requiring context the TPM will not have.

### Improvements to Existing Skills
Good: failure mode examples, edge case clarification, better context handoff between workflow stages, error fixes.

Avoid: removing opinionated guidance, adding steps without judgment value.

### Feedback and Suggestions
Open an issue: what tasks cost you the most overhead, which skills are most/least useful, what is missing from the execution lifecycle.

---

## How to Submit

1. Fork the repo
2. Create a branch: `git checkout -b skill/your-skill-name` or `fix/what-you-fixed`
3. Add or modify files in `skills/`, `agents/`, or `workflows/`
4. Open a pull request with a short description of what it does and why it belongs
